## 一、项目背景与目标

### 1.1 业务背景
情绪音乐盒面向需要快速表达、识别和调节当下情绪的普通用户。用户在日常学习、工作或休息时，常常难以准确描述自己的情绪，也缺少一个低门槛的音乐化反馈工具。本项目通过声音对话和面部识别辅助判断用户情绪，再将情绪转化为音乐参数、短音景和动态可视化，形成一个可展示、可交互的初赛 Demo。产品重点不是替代专业心理咨询，而是提供轻量、有趣、可感知的情绪陪伴体验。

### 1.2 项目目标
在初赛周期内完成一个可在线体验的前端 Web Demo，支持用户通过声音对话和面部识别生成情绪结果，并基于结果推荐音乐参数、播放短音景和展示动态可视化。Demo 需要在主流桌面浏览器中稳定运行，核心流程从输入到生成结果不超过 30 秒。项目不依赖后端账号系统，不接入真实音乐版权内容，优先保证评审可访问、可截图、可理解。

### 1.3 成功指标
- 核心体验完成率 ≥ 90%：用户能从输入情绪到生成音乐卡完成闭环。
- 情绪生成耗时 ≤ 30 秒：从开始识别到结果页展示的平均时间不超过 30 秒。
- 首屏可交互时间 ≤ 3 秒：在常规网络环境下页面快速进入可操作状态。
- 移动端与桌面端关键布局无重叠：至少覆盖 375px、768px、1440px 三类宽度。
- 初赛提交素材完整：体验地址、产品简介、创作思路、TRAE 实践过程、≥3 张截图、≥3 个 Session ID 均可提交。

## 二、用户角色与用户故事

### 2.1 用户角色
| 角色 | 描述 | 权限范围 |
|---|---|---|
| 体验用户 | 打开 Demo 的普通用户或评审 | 使用情绪识别、生成音乐卡、播放短音景、查看本地历史 |
| 创作者 | 项目参赛者 | 配置演示内容、整理截图、说明 TRAE 实践过程 |
| 评审 | 初赛作品查看者 | 访问公开体验地址，查看核心交互和产品完整度 |

### 2.2 用户故事（至少 3 条，每条 P0 必做）
格式：**作为** <角色>，**我想要** <功能>，**以便** <价值>

- **US-001**（P0）作为体验用户，我想要通过声音对话描述当前状态，以便系统理解我的情绪线索。
- **US-002**（P0）作为体验用户，我想要通过摄像头进行面部情绪识别，以便获得更直观的情绪判断结果。
- **US-003**（P0）作为体验用户，我想要获得 AI 推荐的音乐参数和短音景，以便把当前情绪转化为可听见的反馈。
- **US-004**（P0）作为体验用户，我想要看到动态音乐可视化和情绪音乐卡，以便快速理解系统生成结果。
- **US-005**（P1）作为体验用户，我想要保存最近生成的情绪卡，以便回看自己的情绪变化。
- **US-006**（P1）作为评审，我想要无需注册即可体验完整流程，以便快速判断作品质量。

## 三、功能需求清单

| 编号 | 名称 | 描述 | 优先级 | 关联用户故事 |
|---|---|---|---|---|
| F-001 | 声音对话输入 | 支持用户点击录音或输入对话文本，采集“我现在感觉……”这类情绪描述 | P0 | US-001 |
| F-002 | 面部情绪识别 | 调用浏览器摄像头采集画面，并给出开心、平静、焦虑、低落、愤怒、期待等情绪判断 | P0 | US-002 |
| F-003 | 情绪融合判断 | 综合声音对话文本、面部识别结果和用户手动修正，生成最终情绪与强度 | P0 | US-001、US-002 |
| F-004 | AI 音乐参数推荐 | 根据最终情绪推荐 BPM、调式、音色、节奏密度、氛围标签和虚拟曲目清单 | P0 | US-003 |
| F-005 | 短音景播放 | 使用浏览器音频能力生成 10-20 秒短音景，支持播放、暂停、重新生成 | P0 | US-003 |
| F-006 | 动态可视化 | 展示与情绪匹配的声波、粒子或专辑封面动效 | P0 | US-004 |
| F-007 | 情绪音乐卡 | 汇总情绪名称、强度、音乐参数、推荐文案和虚拟曲目，形成可截图的结果卡 | P0 | US-004 |
| F-008 | 本地历史记录 | 使用 localStorage 保存最近生成的情绪卡，支持回看和再次播放 | P1 | US-005 |
| F-009 | 无登录体验 | 用户无需注册登录即可完成全部核心流程 | P1 | US-006 |
| F-010 | 权限降级提示 | 摄像头或麦克风不可用时，提供文本输入和手动情绪选择作为替代路径 | P1 | US-001、US-002 |

## 四、非功能需求

| 类别 | 要求 |
|---|---|
| 性能 | 首屏可交互时间 ≤ 3 秒；情绪结果生成平均耗时 ≤ 30 秒；交互动效保持流畅无明显卡顿 |
| 安全 | 摄像头和麦克风权限仅在用户主动触发时请求；音视频数据不上传服务器；本地记录仅保存在用户浏览器 |
| 兼容性 | 支持 Chrome / Edge / Safari 最新两个大版本；摄像头与麦克风不可用时仍可通过文本和手动选择完成体验 |
| 可维护 | 情绪映射、音乐参数、文案模板与 UI 组件分离；核心生成逻辑可单独测试 |
| 可用性 | 无需登录；核心操作步骤不超过 4 步；结果页必须适合截图提交 |

## 五、范围外说明（明确不做什么）

> ⚠️ 这一节是防止后期争议的关键，必须写！

- ❌ **不做**真实 AI 音乐完整生成，本期只生成短音景、节奏和可视化。
- ❌ **不做**真实版权音乐播放，不接入网易云、QQ 音乐、Spotify 等平台。
- ❌ **不做**账号登录和云端同步，所有历史记录只保存在本地浏览器。
- ❌ **不做**医疗诊断或心理咨询结论，情绪识别结果仅用于娱乐和自我觉察。
- ❌ **不做**移动端原生 App，本期只做响应式 Web Demo。
- ❌ **不做**复杂后台管理系统，初赛阶段不提供运营后台。

## 六、术语表

| 术语 | 英文 | 说明 |
|---|---|---|
| 情绪识别 | Emotion Recognition | 根据声音对话、面部表现或用户选择判断当前情绪类别 |
| 声音对话 | Voice Conversation | 用户通过语音或文本描述心情，系统从内容中提取情绪线索 |
| 面部识别 | Facial Recognition | 通过摄像头画面分析面部表情，辅助判断情绪状态 |
| 音乐参数 | Music Parameters | BPM、调式、音色、节奏密度等用于描述音乐风格的结构化信息 |
| 短音景 | Short Soundscape | 由浏览器实时合成的短时长声音片段，用于表达情绪氛围 |
| 情绪音乐卡 | Emotion Music Card | 汇总情绪判断、音乐参数、可视化和推荐内容的结果展示卡片 |
| 本地存储 | localStorage | 浏览器提供的本地数据保存能力，不依赖账号和服务器 |
